Wiener Linien API - Version 2.0

Dies ist eine inoffizielle Dokumentation für eine API zu den Fahrplan- und Echtzeitdaten der Wiener Linien. Diese API läuft über die Server von fluidtime und wird von den qando-Applikationen genutzt. Für diese API ist keine Lizenz bekannt, auch ist unbekannt, ob diese oder eine ähnliche API im Rahmen von Open Government Data Wien (OGD) jemals veröffentlicht wird.

Mittels der explorer.php kann man recht leicht die API testen. Per Kommandozeile kann man mittels curl -d <xml-daten> http://webservice.qando.at/2.0/webservice.ft die API ansprechen. (Unter OS X kann man noch ein | xmllint --format - zur besseren Übersicht anhängen).

Historisches

Ursprünglich war diese API eine REST-API, seit Version 2.0 (=Version 3.0 der qando Clients) wird im Request ein XML-Dokument geschickt und ein XML-Dokument wird auch wieder empfangen.

API-Endpunkt

POST-Requests werden an die URL http://webservice.qando.at/2.0/webservice.ft geschickt.

Orte

Von/Zu kann in verschiedenen Weisen angegeben werden, die durch den Parameter type spezifiziert werden. Bei einer Von-Angabe, werden die ganzen Parameter mit einem from davor, bei Zu-Angaben mit einem to davor abgewandelt (also z.B. fromType).

type=stop

Parameter Beispiel-Wert Vermutete Bedeutung
name 60201040 Haltestellen-Nummer (dies ist z.B. Praterstern)

Anmerkung: Diese Haltestellennummer kann z.B. mittels der API-Funktion api_search_location_stops_nearby oder api_search_location gefunden werden.

type=coords

Parameter Beispiel-Wert Vermutete Bedeutung
coordName WGS84 Format bzw. Bezugssystem der Koordinaten
wgs84Lat 48.22 Latitude
wgs84Lon 16.39 Longitude

Sonstige Infos

Bei jedem Request muss ein XML-Tag <client> mitgeschickt werden, der zumindest den Parameter clientId enthalten muss. Offenbar wird der Inhalt dieses Tags derzeit nicht geprüft.

API-Funktionen

↓ api_search_location_stops_nearby Sucht Haltestellen in der Nähe einer Koordinate.
↓ api_search_location Sucht nach Haltestellen oder Punkten anhand eines Suchworts
↓ api_get_monitor Abfrage von Echtzeit-Daten für eine Haltestelle
↓ api_get_route Anfrage nach einer Route zwischen from und to.

api_search_location_stops_nearby

Sucht Haltestellen in der Nähe einer Koordinate.

Beispiel-Anfrage

<?xml version="1.0" encoding="UTF-8"?> 
<ft> 
    <request clientId="123" apiName="api_search_location_stops_nearby" apiVersion="2.0"> 
        <client clientId="123"/> 
        <requestType>api_search_location_stops_nearby</requestType> 
        <outputCoords>WGS84</outputCoords> 
        <fromCoordName>WGS84</fromCoordName> 
        <fromType>coords</fromType> 
        <fromWgs84Lat>48.22</fromWgs84Lat> 
        <fromWgs84Lon>16.39</fromWgs84Lon> 
    </request> 
</ft>
